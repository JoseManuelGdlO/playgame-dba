# Etapa 6: Propuesta de Valor

**Estado:** Aprobado
**Fecha:** 2026-07-11

## Frase de propuesta de valor

> "El único juego que te hace querer ser bueno en SQL de verdad, porque ser bueno en SQL de verdad es literalmente cómo mejoras en el juego."

## Frente a plataformas de aprendizaje de SQL (SQLZoo, LeetCode/HackerRank, DataCamp, Codecademy)

| Ellos | Query Path |
|---|---|
| Ejercicios aislados sin contexto ni consecuencia | Cada query tiene un ticket, un jefe, una consecuencia narrativa |
| Progresión = barra de lecciones completadas | Progresión = carrera diegética (rangos, empresas, salario, reputación) |
| Cero sistema de build/rejugabilidad | Sistema de perks/loadout tipo Balatro: cada query resuelta alimenta una build persistente |
| Motivación extrínseca (aprender SQL para el trabajo) | Motivación intrínseca (ver qué pasa/subir de rango/optimizar la build) — el aprendizaje es colateral |
| **Debilidad que no podemos igualar:** cobertura pedagógica exhaustiva, certificaciones, foros | No competimos ahí — no somos alternativa a un curso formal, somos un complemento lúdico |

## Frente a juegos de puzzles de programación (TIS-100, Human Resource Machine, Exapunks)

| Ellos | Query Path |
|---|---|
| Lenguaje ficticio/esotérico inventado para el juego | SQL real — la habilidad se transfiere 1:1 al mundo laboral real |
| Fantasía: ingeniero anónimo resolviendo acertijos abstractos | Fantasía: carrera, jefe, colegas, empresa — contexto humano |
| Público nicho de amantes de puzzles de lógica pura | Diseñado para enganchar primero a fans de sims/tycoon (mercado más grande, Etapa 3) |

## Frente a juegos de gestión/tycoon (Game Dev Tycoon, Software Inc, Two Point Hospital)

| Ellos | Query Path |
|---|---|
| El "trabajo" es abstracto/simulado (barras de progreso, números que suben solos) | El "trabajo" es una habilidad real que el jugador ejecuta (escribe la query de verdad) |
| Sin habilidad transferible al terminar el juego | Al terminar, el jugador sabe SQL de verdad — mayor valor percibido a largo plazo y mejor boca a boca |
| Profundidad de sistemas económicos interconectados | Profundidad del sistema de build-crafting (perks/maestría) sobre una habilidad real — más barato de producir para un solo dev (Pilar 5) |

## Riesgo de la propuesta de valor (a vigilar en producción)

Prometer "SQL real" es también la mayor exposición: si el contenido técnico se siente falso, impreciso o mal calibrado en dificultad, se pierde justo lo que distingue al juego de un tycoon genérico. Mitigación: **Etapa 17 (Sistema de validación SQL)** con un motor de evaluación técnicamente honesto, y **Etapa 16 (Diseño de bases de datos)** con esquemas realistas por industria.
