# Etapa 4: Pilares del Juego

**Estado:** Aprobado
**Fecha:** 2026-07-11

## 1. SQL real, consecuencias reales

Toda query se ejecuta contra un motor SQL de verdad. La corrección, el rendimiento y las buenas prácticas se evalúan genuinamente, nunca se simulan ni se "hacen trampa" narrativamente.

**Filtro de decisión:** si una feature requiere fingir que algo funciona en vez de ejecutarlo de verdad, se descarta.

## 2. La build es tuya (no la query)

La repetición de "escribir SQL" se compensa con profundidad de build-crafting (perks, loadout limitado, combos, maestría por arquetipo — Etapa 1), no con variedad forzada de la acción base.

**Filtro de decisión:** si el juego se siente repetitivo, la solución es más profundidad de build, no más tipos de puzzle artificialmente distintos.

## 3. Carrera, no examen

Todo se presenta a través de la fantasía diegética de una carrera profesional real, nunca como "lección" o "ejercicio". El humor corporativo satírico es la envoltura que hace tolerable la repetición y vende el juego a quien no busca aprender SQL.

**Filtro de decisión:** si algo suena a interfaz de curso (barras de progreso de "lección 3/10", mensajes de "¡Correcto!" genéricos), se rediseña para que suene a jefe, ticket, Slack corporativo o consecuencia laboral.

## 4. Presión de gestión, no presión de mecanografía

El estrés del juego viene de priorizar y gestionar (cola de tickets, SLAs, tiempo de juego), nunca de escribir código contra un cronómetro real.

**Filtro de decisión:** cualquier mecánica que castigue la velocidad de tecleo real del jugador viola este pilar y se descarta (salvo eventos especiales opcionales de clímax, ya acordados como excepción puntual en la Etapa 2).

## 5. Sistemas generativos, no contenido artesanal (pilar de producción)

Dado que es un proyecto de un solo desarrollador apoyado en IA, todo el contenido (esquemas, tickets, empresas, diálogos) debe diseñarse como sistemas y plantillas parametrizables, no como piezas escritas a mano una por una.

**Filtro de decisión:** si una feature solo escala contratando más gente para escribir contenido a mano, se rediseña como sistema generativo o se descarta del alcance.
